Deep Dive

Pipeline Architecture

Each phase transforms data and feeds into the next. The entire pipeline runs in ~8 minutes per execution.

📥 Phase 1: Fetch

Collects customer feedback from all sources for the last 90 days. Reviews are cached locally (12h TTL) to avoid redundant API calls. Each source uses platform-specific queries.

~2,900 total reviews across 3 platforms

🧠 Phase 2: Analyze

Sends reviews to Claude Sonnet with a structured prompt. Claude returns JSON with sentiment scores, topic clusters (5-15), severity ratings, representative quotes, and source breakdowns.

Two windows analyzed: 90 days (landscape view) and 15 days (recent trends). Previous 15 days also analyzed for week-over-week comparison.

~9 Claude API calls per full run

🔗 Phase 3: Correlate

Searches Azure DevOps for existing bugs that match each topic cluster. Filters by platform area path and recency (bugs updated within the time window).

Keywords extracted from cluster topics. Cross-platform terms filtered out. Results include bug state, assignee, and last activity date.

~315 ADO bugs linked across all platforms

📊 Phase 4: Report

Generates a self-contained HTML dashboard (no external dependencies) with platform tabs, time period segments, per-cluster sparklines, ADO links, and "Create Bug" buttons.

Also generates a Markdown report for documentation. Output is ~250KB HTML that works in any browser and can be hosted on SharePoint.

Smart Caching

How Caching Works

The pipeline uses a two-layer caching strategy to minimize API calls and ensure fast, cost-effective daily runs.

💾

Layer 1: Source Cache

Raw reviews from App Store, Play Store, Reddit, and MS Q&A are cached as JSON files with a 12-hour TTL. If the cache is fresh, the Fetch phase completes in <1 second instead of ~2 minutes.

cache/appstore_ios_20260306.json
cache/playstore_android_20260306.json
cache/reddit_mac_20260306.json
12h TTL JSON on disk --no-cache to bypass
🧠

Layer 2: Analysis (No Cache)

Claude analysis runs fresh every time to capture evolving trends. This is intentional — the same reviews may shift in importance as new patterns emerge. ~9 API calls per run (~$1.50).
